Feature Flag Admininstration

Overview

Built and tested with:

  • Oracle OpenJDK version 18 with Spring-boot, Lombok
  • Oracle OpenJDK version 18 with Spring-boot and lombok
  • Gradle
  • Does not use spring profiles for dev/production configuration, current configuraiton uses an in memory H2 RDS
  • Some data is populated by resources/data.sql
  • Uses some Java constructs requiring JDK <15> i.e. record for immutable data containers
  • APIS exist to add/query Customer, Feature and CustomerFeature
  • Archiving features has not been implemented due to time constraints.

Usage:

  • Customner: Lists two customers on initial load
  • Feature: Lists four unique features
  • CustomerFeature: Join Entity/Table that relates features to Customers
    • Seemed a reasonable initial approach for flexibility
    • User: 1, Feature 4: is expired (expired date in the past)
    • Provides simple form to query a feature configuration for a customer.
